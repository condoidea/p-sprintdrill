# 西検4級 追い込みドリル — 基本仕様書 (SPEC v1.0)

## 0. 背景と目的

- 対象: スペイン語技能検定(西検)4級の受験者(試験まで約2週間)
- 学習者プロファイル: 文法は参考書で一周済み。網羅的なレッスンは不要で、**穴の発見と定着**が目的
- 2モジュール構成: **動詞活用ドリル** と **数字ドリル(〜1,000,000)**
- 設計思想: 「教えてから出題」ではなく **「出題して、つまずいた瞬間にだけ教える」**(誤答駆動ミニ解説 + 間隔反復による再出題)

## 1. 技術要件

- 純粋な HTML / CSS / JavaScript の SPA。ビルドステップなし(単一 `index.html` + `js/` + `data/` 構成可)
- フレームワーク不使用(vanilla JS)。ES Modules 可
- 完全オフライン動作。外部API・外部CDNに依存しない
- 進捗永続化: `localStorage`(自前ホスト前提なので使用可)
- 数字モジュールの読み上げに **Web Speech API (speechSynthesis)** を使用。`lang: "es-ES"` の voice を優先選択し、無い場合は読み上げモードを非表示にフォールバック
- スマホ・PC両対応のレスポンシブ

## 2. アーキテクチャ: 共通エンジン + モジュールプラグイン

コアエンジンは出題対象を関知しない。各モジュールは以下のインターフェースを実装して登録する。

```js
// モジュールが実装するインターフェース(概念)
const ModuleInterface = {
  id: "verbs" | "numbers",
  generateItems(): Item[],            // 学習項目の全列挙
  answerFor(item): Answer,            // 正解の生成(文字列 + 構造情報)
  criticalZone(item, answer): Zone,   // ヒント段階制御用の急所定義
  classifyError(item, input, answer): ErrorTag[],  // 誤答分類
  explanationFor(errorTags, item): string,         // ミニ解説(テンプレ差し込み)
  distractorsFor(item, answer): string[],          // 4択モード用ダミー3つ
};
```

### 2.1 Item(学習項目)

```js
Item = {
  id: string,            // 例: "verbs:dormir:pret:3sg" / "numbers:L3:777"
  moduleId: string,
  payload: object,       // モジュール固有(動詞: {inf, tense, person} / 数字: {value, mode})
  tier: 1|2|3,           // 出題重みの初期値に使用(数字はレベルをtierに写像)
}
```

### 2.2 Answer(正解の構造)

```js
Answer = {
  text: "durmió",
  segments: [            // ヒント開示の単位
    { text: "durm", role: "stem",   critical: true  },
    { text: "ió",   role: "ending", critical: false },
  ],
  // 数字の場合は語単位: [{text:"setecientos",critical:true},{text:"setenta",...},{text:"y",...},{text:"siete",...}]
}
```

## 3. 状態遷移(項目ごと)

```
未学習 ──初回出題──▶ 出題中
  出題中: ノーヒント正解(初回) ────────────▶ 習得済み(参考書既習の知識は即抜ける)
  出題中: 誤答 / 「わからない」/ ヒント使用正解 ─▶ 学習中(復習キュー入り)
  学習中: ノーヒント正解を2回(間隔条件を満たして) ▶ 習得済み
  習得済み: 誤答 ─▶ 学習中に戻す
```

- **補助付き正解**: ヒントを1段でも使った正解は「正解」とカウントしない(復習キューに残す)。記録上は `assisted: true`
- 寛容モード(設定でON/OFF): アクセント記号のみの誤りは「△(半正解)」— その場では正解扱いで進むが、復習キューには入れる

## 4. 出題スケジューラ

1. **復習キュー優先**: 期限到来(dueAt <= now)の「学習中」項目を最優先で出題
2. **セッション内再出題**: つまずいた項目は同セッション内で 3〜5問後 に再出題(挿入位置はランダム±1)
3. **セッション終了時テスト**: セッションでつまずいた項目をまとめて再テスト
4. **翌日再テスト**: 前日につまずいた項目は翌日のセッション冒頭に出題(dueAt = 翌日)
5. **習得条件の間隔**: 「ノーヒント正解2回」は同一セッション内の連続では不可。最低でもセッションをまたぐ(または4時間以上空ける)こと
6. **新規項目の混入**: 復習が無い枠に tier 重み付きランダムで未学習項目を投入。初期重み tier1:tier2:tier3 = 5:3:1
7. **弱点重み**: テーマ別(動詞: 時制×動詞タイプ / 数字: レベル)正答率を集計し、正答率が低いテーマの新規投入確率を引き上げる

## 5. ヒント機構(段階式・決め打ち)

ヒントボタンは何度でも押せるが、段階が進むだけ。**急所(critical zone)から一番遠い所から開く**。

| 段階 | 内容 |
|---|---|
| H1 | 文字数マス表示 `_ _ _ _ _ _`(数字モジュールは語数 `___ ___ ___`) |
| H2 | 非クリティカルなsegmentを開示(語幹が急所なら語尾を見せる: `_ _ _ _ ió`) |
| H3 | クリティカルzoneの先頭1文字を開示。次に押すと「わからない」と同じ扱い(解説表示) |

- H1でのアクセント位置の示唆(アクセント付きマスの強調)は **寛容モードON時のみ**
- **習得度によるヒント制限**: 「学習中」で補助付き正解を2回以上経た項目は、H1/H2をスキップして最初からH3相当のみ(足場の漸減)
- ヒント使用は段階ごとに記録(`hintsUsed: [1,2]`)

## 6. 「わからない」ボタン

- 全問に常設。押すと誤答扱い + 即ミニ解説表示 + 復習キュー入り
- 当てずっぽう正解による誤った「習得済み」化を防ぐのが目的。UI上は誤答ボタンより目立たせず、しかし押しやすく

## 7. 判定・誤答分類・ミニ解説

### 7.1 判定

- 入力の正規化: 前後空白除去、小文字化。アクセント記号は **正規化しない**(判定対象)
- 正解と入力の diff(segment単位 + 文字単位)から ErrorTag を1つ以上付与

### 7.2 ErrorTag(動詞モジュール)

| tag | 検出ロジック例 | ミニ解説テンプレの方向性 |
|---|---|---|
| `accent_missing` | アクセント除去後に一致 | 「この形のアクセント位置は〜。点過去1/3人称単数は語末にアクセント」 |
| `stem_change_missed` | 規則活用形(語幹変化なし)を入力 | 「e→ie のブーツ型。nosotros/vosotros では変化しない」 |
| `stem_change_overapplied` | nosotros等で語幹変化を適用 | 「ブーツの外(nosotros/vosotros)は原形語幹」 |
| `pret_3rd_change_missed` | -ir語幹変化動詞の点過去3人称で e>i / o>u 漏れ | 「-ir の語幹変化動詞は点過去3人称のみ e→i / o→u(ブーツ型とは別ルール)」 |
| `wrong_person` | 同時制の別人称の正解形と一致 | 「入力は◯人称の形。求められているのは〜」 |
| `wrong_tense` | 別時制の正解形と一致 | 「入力は◯◯時制の形。〜」 |
| `strong_pret_missed` | 強変化点過去(tuv-等)を規則形で活用 | 「tener の点過去は語幹 tuv- + 無アクセント語尾(-e, -iste, -o…)」 |
| `spelling_change_missed` | busqué→buscé 等 | 「c→qu / g→gu / z→c は音を保つための正書法変化(点過去1人称単数・接続法)」 |
| `subj_stem_wrong` | 接続法を不定詞語幹から作った | 「接続法現在は直説法1人称単数(tengo)の語幹から: tenga」 |
| `participle_irregular_missed` | hacido 等 | 「hacer の過去分詞は hecho(不規則)」 |
| `imperative_polarity` | 否定命令を直説法/肯定形で | 「否定命令は接続法現在を使う: no hables」 |
| `other` | 上記いずれにも該当しない | 正解形 + その項目の生成規則1行 |

### 7.3 ErrorTag(数字モジュール)

| tag | 例 |
|---|---|
| `y_misuse` | y は十の位と一の位の間のみ(ciento y uno は誤り / treinta y uno は正) |
| `cien_ciento` | ちょうど100は cien、101〜199は ciento |
| `irregular_hundreds` | 500 quinientos / 700 setecientos / 900 novecientos(×cincocientos 等) |
| `accent_missing` | dieciséis, veintidós, veintitrés, veintiséis |
| `mil_un` | 1000 は mil(× un mil)。100,000 は cien mil |
| `millon_form` | 1,000,000 は un millón(millón にはアクセント)。複数は millones |
| `fusion_split` | 16〜29は一語(dieciséis)、31以上は分かち書き(treinta y uno) |
| `word_order` / `other` | 語順・その他 |

### 7.4 ミニ解説

- 表示は誤答・「わからない」直後にインラインで 1〜3行。汎用活用表は出さない
- `data/explanations.json` のテンプレに、動詞名・時制・該当形を差し込んで生成
- 解説末尾に同タグの再出題予告(「あとでもう一度出します」)を一言

## 8. 出題モード

### 8.1 入力式(デフォルト)

問題文(例: 「dormir / 点過去 / él」+ 日本語の意味)→ テキスト入力。**アクセントソフトキー**(á é í ó ú ñ ü)を入力欄直下に常設。

### 8.2 4択(本番形式)モード

西検はマークシートのため、4択モードを用意。ダミーはアルゴリズム生成:

- 動詞: ①人称ズラし ②時制ズラし ③語幹変化を忘れた形 ④アクセント除去形 から3つ(正解と重複・相互重複を排除)
- 数字: ①y の位置誤り ②cien/ciento 取り違え ③不規則百位の規則化(cincocientos) ④アクセント除去 から3つ
- 4択モードの正解は習得判定上「ノーヒント正解」と同等に扱うが、習得済み化には入力式での正解を最低1回要求する(設定で変更可)

### 8.3 数字専用モード

1. 数字 → 綴り入力(生成練習)
2. 綴り → 数字入力(読解・高速回転用)
3. **読み上げ → 数字入力**(speechSynthesis es-ES)。再生ボタン・速度0.7/1.0切替

## 9. UI構成

1. **ホーム = 知識マップ**: モジュールごとにグリッド表示(動詞: 時制×動詞タイプのマトリクス / 数字: レベル1〜5)。セルの色 = 未学習(グレー)/学習中(黄)/習得済み(緑)。セル内に項目数と習得率
2. **カウントダウンとノルマ**: 設定した試験日までの残り日数を常時表示。「未習得項目数 ÷ 残り日数」から今日のノルマ(項目数)を算出し、進捗バー表示
3. **セッション画面**: 1セッション = 既定20問(設定可)。問題、入力欄/選択肢、ヒントボタン、「わからない」ボタン、進捗
4. **セッションサマリ**: 正答率、つまずきテーマ、再テスト導線
5. **設定**: 試験日、寛容モード、セッション長、4択/入力比率、データリセット(確認2段階)

## 11. 動詞モジュール仕様

### 11.1 出題範囲(時制・法)

直説法: 現在 / 点過去 / 線過去 / 未来 / 過去未来 / 現在完了(haber + 過去分詞)
接続法: 現在
命令法: 肯定 tú・usted / 否定 tú・usted
任意(設定でON): 現在分詞(estar + gerundio)

人称: yo, tú, él/ella/usted, nosotros, vosotros, ellos/ellas/ustedes(vosotros は設定で除外可)

### 11.2 活用エンジン

フル活用表をデータに持たず、**規則エンジン + メタデータ上書き** で全形を導出する:

1. 規則活用(-ar/-er/-ir × 全時制)の生成関数
2. `verbs.json` の features による上書き: 語幹母音変化(ブーツ型 + -ir点過去3人称)、yo不規則形、強変化点過去語幹(+無アクセント語尾)、未来/過去未来語幹、不規則過去分詞、命令tú不規則、正書法変化(c→qu, g→gu, z→c, gu→g)、y挿入(leer/oír/construir型)、アクセント語幹(enviar/continuar型)
3. `full_override: true` の動詞(ser, ir, estar, haber, dar, ver, saber)は **全活用表をコードに直書き**
4. 接続法現在は「直説法yo形の語幹 + 反対活用語尾」で導出し、不規則6種(sea, vaya, esté, dé, sepa, haya)を上書き
5. 線過去は ser(era), ir(iba), ver(veía) のみ不規則
6. 否定命令 = no + 接続法現在。肯定usted = 接続法現在

**検証**: `verbs.json` 内の `validation_cases` を単体テストとして実装し、全ケース合格を完了条件とする。実装時に1ケースでも食い違ったら、実装側を疑う前に必ず信頼できる活用表と照合して報告すること。

### 11.3 gustar型動詞

gustar / encantar / doler は人称活用ドリルから除外し、専用問題型「(A mí) ___ gusta(n) ...」(間接目的語代名詞 + 3人称単複の選択)として出題する。

### 11.4 知識マップの軸

行 = 時制・法、列 = 動詞タイプ(規則ar/er/ir、語幹変化 e>ie / o>ue / e>i、yo不規則、強変化点過去、完全不規則、正書法変化)。tier3規則動詞はマップ上1セルに集約。

## 12. 数字モジュール仕様

### 12.1 レベル構成(レベル解放はしない。tier写像のみ)

| Lv | 範囲 | 急所 |
|---|---|---|
| 1 | 0〜15 | 個別暗記 |
| 2 | 16〜99 | 16〜29の一語綴りとアクセント / 31以上の y |
| 3 | 100〜999 | cien/ciento、不規則百位(500/700/900)、性一致 |
| 4 | 1,000〜999,999 | mil は無冠詞・無複数、cien mil、veintiún mil 等の語末脱落形 |
| 5 | 1,000,000 | un millón / dos millones / millón de + 名詞 |

### 12.2 number-to-words

JSで `numToWords(n, {gender})` を実装(0〜1,000,000全域)。データファイル不要。`numbers.json` の `test_cases` を単体テストとして全合格を完了条件とする。

### 12.3 出題サンプリング

完全ランダムではなく、**急所を踏む数を優先生成**する: 各レベルから「ルールが発火する数」(例: L3なら x00 / x01 / 5xx / 7xx / 9xx)を重み付きで選ぶ。ルール非発火のランダム数も2〜3割混ぜる。

### 12.4 ヒントの語単位開示

H1 = 語数、H2 = 非クリティカル語の開示(mil は見せて quinientos を隠す等)、H3 = クリティカル語の頭文字。
